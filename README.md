# 代码随想录每日总结



## [209. 长度最小的子数组](https://leetcode.cn/problems/minimum-size-subarray-sum/)

时间复杂度：O(n)

虽然代码里有一个 for 循环嵌套一个 while 循环，但观察 left 和 right 指针：每个元素最多被 right 访问一次，最多被 left 访问一次。两个指针都只是一直向右走，不会回退。所以总操作数是 2n，即 O(n)。

空间复杂度：O(1)

只使用了几个常数级别的变量（left, sum, min_len），没有使用额外的数组或空间。

while循环是对单区间的探索过程

把它想象成一个“吃货毛毛虫”

- **头部 (`right`) 负责吃：** 毛毛虫不停地往前吃（增加 `sum`），直到肚子撑到了（`sum >= target`）。
- **尾部 (`left`) 负责收：** 一旦撑到了，毛毛虫就开始缩尾巴，试图在保证“吃饱”的前提下，让身体变得越短越好。

滑动数组是相反的暴力穷举，或者说暴力穷举的剪枝

💡 深度思考：为什么这道题一定要求正整数？

如果数组中包含**负数**：

1. **滑动窗口会失效**：因为右指针右移不一定让 `sum` 变大，左指针右移也不一定让 `sum` 变小，窗口失去了伸缩的逻辑基础。
2. **二分查找也会失效**：前缀和数组不再是单调递增的（会出现波动），你无法在上面使用二分法。 针对包含负数的情况，通常需要使用**前缀和 + 单调队列**或**哈希表**来处理（如 LeetCode 862 题）。
